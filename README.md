Overview

Komodo Hub is a digital platform designed to support community-driven animal conservation, focusing on endangered species in Indonesia. The system enables schools, communities, and individuals to collaborate, learn, and contribute to conservation efforts through an interactive and secure application.

PROJECT MISSION AND VISION

MISSION
- Mission: To enable safe and scalable community participation in conservation through accessible digital tools and curated learning.
- Measurable objectives: Launch MVP supporting registration, sighting reports, and school libraries
- Success criteria: Functional registration flow; secure student data handling, validated sighting reports.

VISION
- Vision statement:  Bringing endangered species back.
- High level outcome: A nationwide, inclusive platform enabling schools and communities to discover, report, and learn about endangered endemic species.
- Value proposition: Empowering local stakeholders to contribute reliable sightings and educational content while protecting sensitive student data.


Project Objectives
- Promote awareness of endangered species
- Enable schools and communities to participate in conservation programs
- Provide a platform for sharing educational content and wildlife reports
- Ensure secure handling of student and user data
- Create an engaging and user-friendly digital experience

The system supports multiple user types:
Students
- Create accounts and personalise profiles
- Participate in activities and conservation programs
- Upload work (e.g., reports, images, sightings)
Teachers
- Manage classes and student activities
- Communicate via messages and notes
- Monitor student progress
School Admin / Principal
- Register users and manage enrolment
- Generate access codes for students
- Control privacy and visibility
Communities
- Share articles and conservation content
- Publicly display member contributions
System Administrator
- Manage subscriptions and organisations

Features include
Authentication & Access
- Secure login system (PHP backend + MySQL)
- Access code generation for controlled registration
- Role-based access control
Messaging System
- Direct communication between teachers and students
- Notes and feedback on student work
- Internal communication only (privacy-focused)
Profile System
- Customisable user profiles (avatars, themes)
- Student data protected from public access
Public Library
- Shared repository of student and community work
- Public access (without exposing private user data)
Database System
- Relational database structure (MySQL)
Tables include:
- Users
- Schools
- Access Codes
- Messages
- Contents of library such as public library, student findings and public activities.

Desktop Application
- Built using C# (Windows Forms)
- UI includes navigation, user info display, and functional pages
